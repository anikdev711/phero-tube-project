Questions & Answers

1. Discuss the scope of var, let, and const

var is function-scoped. var can be accessible within the function in which it is declared. var is hoisted at the top of the function or global scope. var can be accessible before the declaration in the code. Unlike let and const, var has no block scope. let has block scope. It is only accessible within the block, like if else or loop. let can be hoisted at the top of the block. Before proper initialization of the let variable, it will give a reference error if we want to access it. const has also blocked scope. But the main difference between const and let is that const cannot be reassigned like let. The const variable can be modified if necessary.

2. Tell us the use cases of null and undefined

undefined: A variable that has already been declared but whose value has not yet been assigned is called undefined. There are some cases to get undefined.
1. If the variable is not initialized
2. A function that has no return if necessary
3. Parameters that will not be passed
4. If the return has nothing on its right side
5. A property is used that does not exist in the object.
6. Outside of the index range, accessing the array
7. Deleting elements inside the array
8. A value is set directly to undefined.
null: It indicates the absence of an object value, or value in general. If the value or data is not set, then it is declared null.

3. What do you mean by REST API?

"Representational State Transfer Application Programming Interface" is the full name of the REST API. It is used to create web and mobile applications that connect with servers and receive or deliver JSON or XML data. It is a set of instructions that enables various software applications to communicate with one another via the internet.
